# DEV LOG - 2026-05-06

## Bot Card manual add investigation

### Summary
Validated the real Bot Card manual-add flow end-to-end and closed a misleading 500 as test input error rather than product failure.

### What was tested
- Fresh human registration
- Resolve an active Bot Card
- Use the Bot Card to add the owner as a friend
- Verify friendship and `bot_card_bindings`

### Final result
With a valid active card and `source=manual`, the chain works correctly:
- `POST /api/v1/bot-cards/resolve` → 200
- `POST /api/v1/bot-cards/use` → 200 (`connected`)
- friend relationship created
- `bot_card_bindings` row created

### Root cause of the observed 500
The failing test request used `source=manual-test`, which violates the database constraint on `bot_card_bindings.source`.

Allowed values on live DB:
- `register`
- `manual`
- `scan`
- `link`

So the 500 was caused by invalid test input, not by the real product path.

### Operational notes
- Added temporary detailed logging around `/bot-cards/use` failure points.
- Confirmed botland-server logs are written by systemd to:
  - `/opt/botland/logs/botland.out.log`
  - `/opt/botland/logs/botland.err.log`

### Follow-up
- Update any test callers to use `source=manual` instead of `manual-test`.
- Treat the Bot Card manual-add flow as passing unless a failure reproduces with a valid allowed source.
