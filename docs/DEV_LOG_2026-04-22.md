#### Plugin WS 兼容性修复（关键 Bug）

**问题**: Botland channel plugin（OpenClaw 进程内）使用 Node.js 22 内置 `globalThis.WebSocket`（undici 实现），连接 Botland server（Go gorilla/websocket）时，`open` 事件触发后 ~300ms 立刻 `close code=1006`，导致 agent 永远无法在线。

**排查过程**:
1. Server 日志（`/tmp/botland-server.log`）确认 human 消息到达但 `message stored offline`（agent 不在线）
2. Plugin 日志确认 `connected → closed code=1006` 循环
3. 独立脚本使用同一 token + `ws` npm 库连同一 server → 稳定
4. 确认 nginx WS proxy 配置正确
5. 锁定：Node 22 undici WebSocket 与 gorilla/websocket 兼容性问题

**修复**: 
```js
// Before (broken)
const ws = new WebSocket(wsUrl);  // Node 22 built-in

// After (works)
import WS from "ws";
const ws = new WS(wsUrl);  // ws npm library
```

所有 `WebSocket.OPEN` / `WebSocket.CONNECTING` 引用同步改为 `WS.OPEN` / `WS.CONNECTING`。

`package.json` 新增依赖 `"ws": "^8.0.0"`。

**⚠️ 教训**: Node.js 22+ 内置 WebSocket 和 `ws` 库行为不同。在需要连接 gorilla/websocket 等 Go WS server 时，必须使用 `ws` npm 库。
